# Proposal Template Instructions

## Frontmatter

Update all fields:
- `title`: Your proposal's title
- `proposer`: Your club's slug (e.g., example-club)
- `date`: Submission date (YYYY-MM-DD)
- `requested_amount_usd`: Amount requested from College DAO

## Status Badges

- Use "Draft" status while preparing
- Update to "Submitted" when ready for review
- Do not modify badge formatting, only the status text

## Important Callout

Update the `<Important>` component to show your requested amount and club name.

## 1. Event Overview

Answer each question clearly and concisely:
- **What:** Describe the event/initiative
- **Where:** Location (city, venue, or "Virtual")
- **When:** Specific dates
- **Why:** Explain the value and impact

## 2. Budget & Funding Sources

- First table: List all costs with line items and amounts
- Second table: Show all funding sources with amounts and percentages
- Totals must match between tables

## 3. Timeline

List key milestones with dates. Include:
- Application/planning phase
- Execution phase
- Reporting/follow-up phase

## 4. Give-back Commitment

All funded proposals must commit to returning a percentage of revenue generated.
- If revenue-generating: specify percentage and estimated amount
- If non-revenue-generating: state that unused funds will be returned

## 5. Success Metrics & Proof

- Define measurable outcomes (attendees, connections, projects, etc.)
- List the proof you will submit after the event (photos, receipts, reports)
- Include a deadline for submitting proof
